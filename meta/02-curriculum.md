# Curriculum

Version: 1.0

---

# Overview

The handbook is divided into progressive learning stages.

Each stage builds upon previous knowledge.

Readers should complete the stages in order.

---

# Stage 1 — Engineering Mindset

Objective:

Learn how software engineers think.

Topics:

- What is Software Engineering?
- Programming vs Engineering
- Product Thinking
- AI-First Development
- Engineering Trade-offs
- Technical Debt
- Build vs Buy
- Simplicity
- Maintainability

Learning Outcome:

Think like an engineer instead of a programmer.

---

# Stage 2 — Software Architecture

Objective:

Learn how to structure software before writing code.

Topics:

- Layered Architecture
- Feature-Based Architecture
- Vertical Slice Architecture
- Clean Architecture
- Monolith
- Modular Monolith
- Microservices
- Dependency Injection
- Repository Pattern
- Service Layer
- Event-Driven Architecture

Learning Outcome:

Design maintainable software architectures.
