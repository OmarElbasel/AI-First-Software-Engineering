## قائمة المراجعة

- [ ] مراجعة ذاتية للـ diff بالكامل — بما في ذلك الكود المولَّد — سطرًا بسطر
- [ ] الاختبارات تغطي السلوك الجديد وأنماط فشله، وليس المسار السعيد فقط
- [ ] لا توجد أسرار أو بيانات اعتماد أو مخرجات debug في الـ diff
- [ ] عمليات الترحيل (Migrations) متوافقة مع الإصدارات السابقة، أو أن ترتيب النشر (deploy order) موثَّق
- [ ] تحديث الـ Docs/ADRs عند تغيُّر السلوك أو البنية (architecture)
