# القوالب

نقاط بداية قابلة لإعادة الاستخدام للمستندات الهندسية. انسخ الملف،
واحذف التعليقات التوضيحية، واملأه.

القوالب موجودة لإزالة مشكلة الصفحة الفارغة، لا لفرض البيروقراطية.
كل قسم في كل قالب موجود لأن تجاوزه له نمط فشل معروف. إذا كان
قسم ما لا ينطبق حقًا، فاحذفه — لكن اعلم السبب.

## الفهرس

| Template | Use it when | Do not use it when |
|---|---|---|
| [chapter-technical.md](chapter-technical.md) | كتابة فصل في الدليل حول تقنية أو أسلوب | الفصل يعلّم العقلية أو المنهجية — استخدم القالب المفاهيمي |
| [chapter-conceptual.md](chapter-conceptual.md) | كتابة فصل في الدليل حول العقلية أو الحكم أو المنهجية | الفصل يحتوي على كود حقيقي لعرضه — استخدم القالب التقني |
| [adr.md](adr.md) | اتخاذ قرار مكلف التراجع عنه أو سيُطعن فيه لاحقًا | الاختيار قابل للعكس بشكل trivial (اسم متغير، مكتبة صغيرة) |
| [pull-request.md](pull-request.md) | فتح أي pull request غير trivial | — |
| [project-brief.md](project-brief.md) | بدء مشروع جديد أو ميزة رئيسية، قبل توجيه أي prompt إلى الذكاء الاصطناعي | العمل تغيير صغير ومفهوم جيدًا |
| [claude-md-starter.md](claude-md-starter.md) | إعداد مستودع للتطوير بنهج AI-first | — |

## القواعد

- قالبا الفصل يعكسان قائمة الأقسام المعيارية في
  [`meta/00-CONSTITUTION.md`](../meta/00-CONSTITUTION.md). إذا تباعدا
  يومًا ما، فالدستور هو المرجع.
- تظهر الإرشادات على شكل تعليقات HTML (`<!-- like this -->`) حتى
  لا تُعرض أبدًا. احذف التعليقات أثناء ملء كل قسم.
- تُنسخ القوالب مع إصدارات المستودع. حسّنها عندما يكشف مستند
  حقيقي عن فجوة — لا تشتق نسخًا خاصة.
